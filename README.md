# Research Agent

Research Agent is an advanced AI-powered research assistant designed to automate the complex process of gathering, analyzing, and synthesizing information. It acts as an autonomous agent that plans research strategies, executes web searches, extracts relevant data, and compiles comprehensive reports, all while providing real-time feedback to the user.

## üöÄ Features

- **Autonomous Research Planning**: Deconstructs complex user queries into actionable research steps.
- **Intelligent Web Search**: Leverages the Tavily Search API to find high-quality, relevant sources.
- **Smart Content Extraction**: Scrapes and processes web content to extract key insights, filtering out noise.
- **Vector-Based Memory**: Utilizes Qdrant vector database for semantic search and efficient information retrieval.
- **Comprehensive Report Generation**: Synthesizes gathered data into structured, easy-to-read research reports.
- **Real-Time Progress Streaming**: Keeps users informed with live updates on the agent's actions via Server-Sent Events (SSE).
- **PDF Export**: Generates professional PDF reports of the research findings.
- **Secure Authentication**: Integrated with Clerk for secure user management and authentication.

## ‚öôÔ∏è Installation & Setup

### 1. Clone the Repository

```bash
git clone <repository-url>
cd research-agent
```

### 2. Backend Setup

Navigate to the backend directory and install dependencies:

```bash
cd backend
npm install
```

Create a `.env` file in the `backend` directory with the following variables:

```env
PORT=3001
MONGODB_URI=mongodb://localhost:27017/research-agent
# or your MongoDB Atlas URI

# LLM Configuration
LLM_API_KEY=sk-...
LLM_BASE_URL=https://api.openai.com/v1 # or your provider's URL
LLM_MODEL=gpt-4o # or gemini-1.5-pro, etc.

# Fallback LLM (Optional)
LLM_FALLBACK_API_KEY=...
LLM_FALLBACK_BASE_URL=https://openrouter.ai/api/v1
LLM_FALLBACK_MODEL=...

# Search & Vector DB
TAVILY_API_KEY=tvly-...
QDRANT_URL=http://localhost:6333
QDRANT_API_KEY= # Optional if running locally without auth

# Authentication
CLERK_PUBLISHABLE_KEY=pk_test_...
CLERK_SECRET_KEY=sk_test_...
```

Start the backend server:

```bash
npm run dev
```

### 3. Frontend Setup

Navigate to the client directory and install dependencies:

```bash
cd ../client
npm install
```

Create a `.env.local` file in the `client` directory:

```env
NEXT_PUBLIC_CLERK_PUBLISHABLE_KEY=pk_test_...
CLERK_SECRET_KEY=sk_test_...
NEXT_PUBLIC_API_URL=http://localhost:3001
```

Start the frontend development server:

```bash
npm run dev
```

The application should now be running at `http://localhost:3000`.
